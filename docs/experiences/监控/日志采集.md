# 日志采集

前端日志采集是线上问题排查和数据分析的基础。与性能指标不同，日志更关注**业务层面的用户行为和异常信息**。

## 错误日志

### 采集内容

| 类型 | 描述 |
|---|---|
| JavaScript 运行时错误 | SyntaxError、TypeError、ReferenceError 等 |
| 资源加载失败 | 图片、脚本、样式表等静态资源 404/500 |
| Promise 未捕获异常 | unhandledrejection 捕获的拒绝 |
| 框架层错误 | React ErrorBoundary、Vue errorCaptured |

### 采集方案

```js
// 全局错误监听（JS 运行时错误）
window.onerror = function (msg, url, line, col, error) {
  report({ type: 'error', msg, url, line, col, stack: error?.stack })
  return false // 不触发浏览器默认错误处理
}

window.addEventListener('error', (e) => {
  // 捕获阶段的错误（资源加载失败等）
  report({ type: 'error', msg: e.message, filename: e.filename, lineno: e.lineno })
}, true)

// Promise 未捕获异常
window.addEventListener('unhandledrejection', (e) => {
  report({ type: 'unhandledrejection', reason: e.reason })
})

// 资源加载失败（img、script、link 等）
window.addEventListener('error', (e) => {
  if (e.target !== window) {
    // 资源错误不冒泡到 window，用捕获阶段监听
  }
}, true)
```

### 上报字段

- **错误类型**：SyntaxError、TypeError 等
- **错误堆栈**：包含文件名、行号、列号
- **用户路径**：用户在出错前访问的页面序列
- **环境信息**：UA、屏幕分辨率、网络状态
- **资源 URL**：失败资源的完整地址

## 行为日志

### 采集内容

| 类型 | 描述 |
|---|---|
| PV / UV | 页面浏览量、独立访客数 |
| 停留时长 | 用户在每个页面的驻留时间 |
| 点击热区 | 用户点击位置分布 |
| 页面跳出率 | 仅访问一页就离开的比例 |
| 转化漏斗 | 关键路径的完成率 |

### 采集方案

```js
// 路由监听（PV）
const routes = []
router.afterEach((to) => {
  const lastRoute = routes[routes.length - 1]
  const duration = Date.now() - lastRoute.enterTime

  report({
    type: 'page_view',
    path: to.path,
    referrer: lastRoute.path,
    duration
  })

  routes.push({ path: to.path, enterTime: Date.now() })
})

// 页面可见性变化（离开时长统计）
document.addEventListener('visibilitychange', () => {
  if (document.visibilityState === 'hidden') {
    report({ type: 'page_hidden', path: location.pathname, timestamp: Date.now() })
  }
})

// 事件代理（点击行为）
document.addEventListener('click', (e) => {
  const target = e.target.closest('[data-report]')
  if (target) {
    report({
      type: 'click',
      element: target.dataset.report,
      path: location.pathname
    })
  }
}, true)

// 页面关闭前（确保最后一条日志发出）
window.addEventListener('beforeunload', () => {
  navigator.sendBeacon('/analytics', JSON.stringify({ type: 'leave', path: location.pathname }))
})
```

## 接口日志

### 采集内容

| 类型 | 描述 |
|---|---|
| 接口成功率 | 成功请求 / 总请求 |
| 接口响应时间 | P50 / P95 / P99 分布 |
| 接口状态码 | 2xx / 4xx / 5xx 占比 |
| 接口重试次数 | 自动重试触发次数 |

### 采集方案

```js
// axios 拦截器
const reportQueue = []

axios.interceptors.response.use(
  (response) => {
    reportQueue.push({
      type: 'api_success',
      url: response.config.url,
      method: response.config.method,
      duration: response.config.metadata?.duration,
      status: response.status
    })
    return response
  },
  (error) => {
    reportQueue.push({
      type: 'api_error',
      url: error.config?.url,
      method: error.config?.method,
      status: error.response?.status,
      errorType: error.code // ECONNABORTED、ERR_NETWORK 等
    })
    return Promise.reject(error)
  }
)
```

## 环境日志

### 采集内容

| 类型 | 描述 | 采集方式 |
|---|---|---|
| 浏览器类型/版本 | Chrome、Firefox、Safari 等及版本号 | UA 解析 |
| 操作系统 | iOS、Android、Windows、macOS | UA 解析 |
| 网络类型 | 4G、WiFi、unknown | `navigator.connection` |
| 设备内存 | 低/高内存设备 | `navigator.deviceMemory` |
| 屏幕分辨率 | DPR、屏幕尺寸 | `window.screen` |
| 可用内存 | Web API | `navigator.deviceMemory` |

### 采集方案

```js
const getEnvInfo = () => {
  const ua = navigator.userAgent
  const connection = navigator.connection || {}
  const screen = window.screen

  return {
    ua,
    browser: parseUA(ua).name + ' ' + parseUA(ua).version,
    os: parseOS(ua),
    network: connection.effectiveType, // 4g / wifi / 2g / 3g / unknown
    deviceMemory: navigator.deviceMemory, // GB
    screen: `${screen.width}x${screen.height}`,
    dpr: window.devicePixelRatio,
    language: navigator.language,
    cookieEnabled: navigator.cookieEnabled
  }
}
```

## 日志上报策略

### 批量上报

高频日志（如点击、滚动）不实时上报，先缓存再批量发送：

```js
const logBuffer = []

const flush = () => {
  if (logBuffer.length === 0) return
  const logs = logBuffer.splice(0, logBuffer.length)
  navigator.sendBeacon('/analytics', JSON.stringify(logs))
}

// 每 3s 或缓存满 20 条时上报
setInterval(flush, 3000)
```

### 数据脱敏

上报前对敏感信息做处理：

```js
const sanitize = (data) => {
  const sensitive = /phone|pwd|token|cookie/gi
  return JSON.stringify(data).replace(sensitive, '[filtered]')
}
```
